由“哪些类型的值可以做python dict的key”所引发的血案

发表于 2015-05-30   |  

今天遇到了这么一个问题，在C站的forum里小争论了一下

list/dict/tuple/0/set([])/str中哪些类型能作为dict的key？

平常较常用的dict的key值类型就是int啊，string啊，float啊，以前真没思考过这个问题，不过现在想来还是有点意思。

什么是hashable？

请看官方对hashable给出的解释

hashable

An object is hashable if it has a hash value which never changes during its lifetime (it needs a hash() method), and can be compared to other objects (it needs an eq() or cmp() method). Hashable objects which compare equal must have the same hash value.

Hashability makes an object usable as a dictionary key and a set member, because these data structures use the hash value internally.

All of Python’s immutable built-in objects are hashable, while no mutable containers (such as lists or dictionaries) are. Objects which are instances of user-defined classes are hashable by default; they all compare unequal (except with themselves), and their hash value is their id().

一个object是hashable的有两点要求：

  • 在生命周期内不可变（实现hash()方法）
  • 能进行对象比较（有eq()或cmp()方法）

python中所有不变的内建对象都是hashable的，所有可变的容器（比如，list or dict）都不是hashable。

回顾问题

list/dict/tuple/0/set([])/str中哪些类型能作为dict的key？
上述类型全都是内建类型，所以问题转化成list/dict/0/set([])/str哪些是immutable的？
进一步转化成list/dict/tuple/0/set([])/str哪些有hash()方法？
0 : int有hash()方法，相应的float也有

str : 有hash方法

tuple : 有hash方法

但是list/dict/set([])都是没有hash方法的，是可变的数据类型，不能作为dict的key。

附：解释一下set和tuple的区别

初学者特别容易将set和tuple弄混。所以放上set和tuple的定义，对比出真知。

set

Build an unordered collection of unique elements.

tuple

tuple is an immutable sequence type

翻译过来就是set存储无序的、不重复的元素。自动去重，使用方法是”set([])”。

可以看出，set和tuple实际上是完全不相关的两种数据类型。
