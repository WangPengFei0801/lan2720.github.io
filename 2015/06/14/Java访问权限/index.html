权限的基本介绍

public

类

当两个类不在同一个包中，public权限就显示出作用了。
如果一个类不是public的话，外部包无法对这个类进行访问。

成员变量和函数

如果想在包的外部调用某一个类的成员变量或者成员函数，那么这个成员必须是public，也就是必须是外部包可见的才能被访问，否则无法访问该成员。

总结

可以修饰成员函数、成员变量和类。没有任何限制，外部包也可以访问。

default

在同一个package中的类，变量，函数均可互相访问。但是，不在一个package中的无法访问。

总结

default也叫包访问权限。

private

只能修饰变量、函数，不能用来修饰类。
表示这个变量或者这个函数只能在当前类中使用，而在类的外部通通无法使用，即使处于同一个包也无法访问。

protected

稍微复杂，主要和继承有关系。

访问权限和继承的关系

default

由于default权限的成员仅在同一个包内可以自由访问。
如果子类和父类不在同一个包当中，子类虽然可以继承到父类中default权限的成员，但是没有访问权限。

private

由于private只能在类内访问。
如果父类的成员变量或者成员函数声明为private，其子类是无法访问该成员变量或函数的。

protected

如果子类和父类不在同一个包当中，但是又希望子类能使用父类的某些成员变量或函数，就在父类中将这些成员声明为protected。

protected拥有和default一样的权限，但是该权限只能修饰成员变量和成员函数（不能修饰类）。
除此之外，protected允许跨包继承。

子类可以使用父类中被声明为protected的成员变量和成员函数，即使子类和父类在不同的包当中。

对比protected和public修饰的成员变量和成员函数

protected权限修饰的成员，同包内可自由访问，跨包的话只有继承了父类的子类能访问，其他的任何情况都无法访问到protected修饰的成员。
public修饰的成员任何类都能访问。

权限总结

public > protected > default > private

注意事项

一个类或者一个成员变量或函数的权限应该尽可能的小，才能体现面向对象的封装性。
